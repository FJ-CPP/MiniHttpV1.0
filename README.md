# MiniHttpV1.0

## 前言

HTTP作为应用层应用最广泛的协议之一，是每一个开发者必备的技能。

为了更加深入地理解HTTP协议，同时初步认识前端与后端的对接，我在参考了众多代码和查阅书籍后，完成了本HTTP项目，代码1000+，虽然量不大，收获颇丰。

本项目集合了C++、Linux系统编程等众多知识，基于多线程、套接字编程，作为Linux系统与网络学习阶段的结业项目是一个非常不错的选择。

## 项目介绍

### 一、模块划分

#### 1、TCP模块

封装系统的套接字接口，为HTTP服务器提供TCP连接服务。

#### 2、HTTP Protocol模块

HTTP协议模块，用来接收客户端发送的报文以及根据请求响应对应的资源。

#### 3、线程池模块

基于“生产者消费者模型”。客户端连接请求到来后，将请求封装成任务并加入线程池的任务队列。

#### 4、日志模块

在服务器运行的同时写入日志信息。

#### 5、WEB资源模块

本模块又分为两部分：

##### 1. 网页资源

即html网页。

##### 2. CGI程序(Common Gateway Interface)

即基于公共网关接口协议的应用程序。通过环境变量及管道获取用户的请求数据，再以管道将处理完毕的数据返回给HTTP服务进程。

本项目中提供了表单提交处理程序以及连接后端数据库的用户注册程序。

### 二、功能实现

#### 1、短连接

接收并响应请求后即断开与客户端的TCP连接。

#### 2、请求方法

实现了常用的GET、POST方法，其余方法有待扩展。

#### 3、协议处理

基于HTTP协议处理请求行、请求报头、请求正文，同时封装状态行、响应报头、响应正文进行报文发送。

### 三、可扩展性分析

#### 1、HTTP方面

- 加入更多的HTTP请求方法对应的处理方式，如PUT、DELETE、HEAD。
- 实现长连接：客户端连接socket管理

#### 2、WEB资源方面

- 加入更多的个性化静态网页
- 更多实用的CGI程序
